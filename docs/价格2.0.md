## 1. 变更背景

Pricing 1.0 主要按 **模型渠道差价 + 75% 视频毛利护栏** 定价，**未显式摊入** 固定基础设施成本（服务器、OSS 存储、CDN/出网等）。当前固定月支出约 **¥5,000**，本版在 **不削减月积分总量、不改变 BYOK 含次与超额扣分** 的前提下，上调订阅月费与 BYOK 技术服务费，并将通用轻量包最低起购调整为 **1,500 积分**。

## 11. 调价后扣费全链路核查（Finance 2.0）

| 环节 | 真源 / 入口 | 核对要点 |
|------|-------------|----------|
| 价目 seed | `seed-credit-billing.ts` + `db:post-billing-setup` | `MembershipPlan.priceYuan`、加油包、`pricePerCreditYuan` 与 §2–5 一致 |
| 生成前预检 | `credit-pre-check.ts` | 与 `computeChargeCredits` 同口径（逐档单价 × 单位数） |
| Gateway 扣减 | `gateway-credit-settlement.ts` → `CreditLedger` | 成功日志 `creditsCharged` 写入 `GatewayRequestLog` |
| 扣减明细 | `billing-details-service.ts` → finance-web `/fees/billing/details?tab=charge` | 「平台/消耗积分」= 流水 SETTLE 合计 |
| 用量汇总 | `/fees/usage` · `account-usage-summary.ts` | 七类用量 + 双池余额 |
| Gateway 运维 | gateway-web 日志表 **Credits** 列 | 与 finance-web 同字段 `creditsCharged`（≠ Usage ¥ 厂商参考价） |
| 工具站 | `GET /api/tool-billing-details` · `GET /api/tool-credits` | Finance 2.0 扣减明细 + 双池余额 |
| 存量账户 | `pnpm billing:align-active-pricing --confirm` | 在用客户扣费单价 / BYOK 月费对齐 Pricing 2.0（不补差、不改余额） |

**勿混用**：gateway-web **Usage ¥** = 厂商挂牌参考；**Credits** = 平台代付积分。`CreditAccount` 双池为唯一用户余额口径（legacy `Wallet.balancePoints` / `ToolUsageEvent` 已退役）。

**自动化**：`pnpm test`（103）+ `pnpm test:finance-integration` + scenario-lab 60 行毛利带。
