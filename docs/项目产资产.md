## 原始需求（摘要）

1. **资产类型（11 类）**：角色、场景、道具、大纲、分镜脚本、音频、分镜图、分镜视频、数字人、风格、提示词。
2. **个人 / 团队**：类型相同；个人仅私有，团队可共享（`AssetVisibility`）。
3. **节点顶栏**：所有可入库节点须有「保存为资产」；保存时选择私有或团队共享。
4. **组**：分组后可「保存组为资产」。
5. **账号体系**：对齐现有 personal / team 租户（`14-tenant-team-design.md`）。
6. **三种画布**：Story-Pro 1.0、影视 2.0、分镜视频 1.0 → **共用一套**项目资产 API 与 `/assets` UI。
7. **顶栏样式**：与现 `Pro2ImageNodeToolbar` 一致（`PRO2_IMAGE_NODE_TOOLBAR_*`）。

---

## 方案结论（一句话）

**一套 `ProjectAsset` 表 + 一套 Platform API + 一套 `/assets` 界面**；画布 edition 只决定「从哪种节点导出」；个人/团队用 `tenantId` + `visibility`；同时一人编辑用 **editLease**（与生产 `locked` 分离）。

### 节点 data 快照（2026-06 确认）

| 写入 | 不写入 |
| --- | --- |
| `payload.nodeType` + `payload.nodeSnapshot`（`ossUrl`、提示词 `dockInput`、模型参数、风格 ref 等） | 上游/下游连线、`pro2HubNodeId` / `pro2GroupId` / `referencedNodeIds` |
| `thumbnailUrl` + `refs[]` 媒体槽 | `blobUrl`、上传中状态、`runtime` 临时字段 |
| 组资产 `GROUP_BUNDLE`：`layout.nodes/edges` 整组结构 | 组外游离边 |

插入画布：`insert-map` 合并 `nodeSnapshot` + 封面/refs 兜底 `ossUrl`；不恢复组外连线。
