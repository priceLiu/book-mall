# 云端自动剪辑（Media Render）操作手册

> 技术契约：`book-mall/doc/tech/media-render-timeline-v1.md`  
> 架构说明：`docs/全站架构图与配置表.md` §5（render limits env）

---

## 1. 功能是什么

在 **保留剪映 ZIP 导出** 的前提下，平台在 book-mall 云端用 **ffmpeg** 把多镜视频合成一条 **MP4**：

| 能力 | 说明 |
|------|------|
| 转场 | 默认 **0.6s 交叉淡化**（xfade），可调 |
| 字幕 | 优先用分镜 **台词字段**（非语音识别） |
| 存储 | 仅新增 **1 个** 临时成片 OSS；源镜 **不复制** |
| 保留 | 默认 **7 天限时下载**，到期删 OSS |
| 统一 | 画布、电商合并、Platform API **同一引擎** |

---

## 2. 谁要装 ffmpeg？

| 角色 | 要不要装 | 说明 |
|------|----------|------|
| **终端用户**（画布/电商点「自动剪辑」） | **不用** | 剪辑在 **book-mall 服务器** 跑，用户只上传分镜视频 URL |
| **本地开发者** | **要**（仅 book-mall 进程） | 一键见下 |
| **生产运维** | **不用单独装** | `book-mall` Docker 镜像已内置 ffmpeg |

> 不装 ffmpeg → **自动剪辑不可用**；剪映 ZIP 导出 **不受影响**。

### 2.1 本地开发：傻瓜安装（推荐）

```bash
cd book-mall && pnpm media-render:setup-ffmpeg   # macOS 自动 brew install
pnpm media-render:check-ffmpeg                   # 检查是否可用
```

然后 **重启** `pnpm dev:all`。`dev:preflight` 若未检测到 ffmpeg 会黄色提示上述命令。

手动安装（macOS）：

```bash
brew install ffmpeg
ffmpeg -version && ffprobe -version
```

其它前置：

1. `cd book-mall && pnpm db:deploy`（含 `MediaRenderJob` 表）
2. `.env.local` 中 **OSS_*** 已配置
3. 源镜为 **HTTPS** 公网 URL（画布/电商生成后的 OSS 地址）

### 2.2 生产 / CloudBase

- **book-mall 镜像已含 ffmpeg**（`book-mall/Dockerfile` → `apt-get install ffmpeg`）；勿用未装 ffmpeg 的裸 Node 镜像跑 book-mall。
- 上线后自检：`docker exec <book-mall容器> ffmpeg -version`
- 定时清理过期成片（建议每日 cron / SCF）：
  ```bash
  cd book-mall && pnpm media-render:expire
  ```

## 3. 画布：如何接上工作流

### 3.1 工作流结构

```text
[分镜脚本列] ──┐
               ├──► [剪映导出节点] ──► ① 分镜包 ZIP（保留）
[视频列·各镜] ──┘                   ② 剪映草稿 ZIP（保留）
                                    ③ 自动剪辑成片 MP4（新增）
```

**不需要新节点类型**，沿用现有 **「剪映导出」** / **「剪映导出 · 专业版」** 节点即可。

### 3.2 操作步骤

#### 专业版漫剧

1. 打开画布项目，确认 **影视专业 · 启动** 工作区已展开：
   - **分镜脚本列**：各镜有台词（可选，用于字幕）
   - **视频列**：**每一镜** 已生成 `mp4`（必需）
   - 配音轨可选（自动剪辑当前以视频内音轨为主）
2. 从节点面板拖入 **「剪映导出 · 专业版」**，连到同一工作区（与导出 ZIP 相同连线）。
3. 选中节点，查看「已识别 N 镜」；含视频数应 ≥ 1。
4. 在节点底部 **「云端自动剪辑成片」** 区域：
   - 调整 **转场时长**（默认 0.6 秒）
   - 可选勾选 **烧录台词字幕**
   - 点击 **「自动剪辑成片（MP4）」**
5. 等待进度（通常 1～5 分钟，镜数多会更久）。
6. 完成后点 **「下载成片 MP4」**；注意 **7 天内下载** 提示。

#### 标准版漫剧

步骤相同，使用 **「剪映导出」** 节点；视频来自旧版分镜引擎或工作区视频列。

### 3.3 与剪映 ZIP 的关系

| 按钮 | 用途 | 是否变更 |
|------|------|----------|
| 分包导入 · 分镜包 ZIP | 本地拖入剪映时间线 | **不变** |
| 全包导入 · 剪映草稿 ZIP | Mac 解压到剪映草稿目录 | **不变** |
| 自动剪辑成片 MP4 | 云端合成，限时下载 | **新增** |

可 **先自动剪辑出片预览**，再按需导出 ZIP 进剪映精修。

### 3.4 常见问题

| 现象 | 处理 |
|------|------|
| 「至少需要 1 个视频片段」 | 视频列各镜先生成 mp4 |
| 「云端自动剪辑服务暂未就绪」 | **用户**：稍后再试。**开发**：`cd book-mall && pnpm media-render:setup-ffmpeg` 后重启 dev:all |
| 红字 `Invalid prisma.mediaRenderJob.create` | **开发**：`cd book-mall && pnpm db:deploy && pnpm db:generate`，然后 **重启 pnpm dev:all**（book-mall 须加载新 Prisma Client） |
| 「同时进行的剪辑任务不能超过 2 个」 | 等待当前任务完成 |
| 「成片时长超过上限」 | 减少镜数或缩短单镜（默认总时长 ≤600s） |
| 下载链接失效 | 超过 7 天已清理；需重新提交剪辑 |

## 4. 电商分镜：合并分镜视频

1. 微剧情分镜项目中，至少 **2 镜** 已生成分镜视频。
2. 点击 **「合并分镜视频」**（助手或交付区均可）。
3. 系统走云端渲染（含转场），**异步合成**；界面显示「云端合成…」。
4. 成功后提示 **7 天内可下载**；交付快照写入 `renderJobId`、`renderExpiresAt`。
5. **不再** 默认永久写入视频库；长期保留需购买容量包后 **pin**（P2）。

## 5. 参数说明（RenderProfile）

| 参数 | 默认 | 说明 |
|------|------|------|
| `transition.type` | `xfade` | `xfade` 交叉淡化；`none` 硬拼接 |
| `transition.durationSec` | `0.6` | 转场秒数，建议 0.4～0.8 |
| `subtitle.mode` | `script` | 用分镜台词生成 SRT 时间轴 |
| `subtitle.burnIn` | `false` | `true` 时烧录字幕到画面 |
| `video.scaleMode` | `fit1080p` | 统一 1080p，避免规格不一致合成失败 |

API 调用方可 POST `profile` 覆盖；详见 `book-mall/doc/tech/media-render-timeline-v1.md`。

## 6. 限额（管理员 env）

| 变量 | 默认 |
|------|------|
| `MEDIA_RENDER_MAX_CLIPS` | 30 |
| `MEDIA_RENDER_MAX_OUTPUT_DURATION_SEC` | 600 |
| `MEDIA_RENDER_MAX_SOURCE_BYTES_PER_CLIP` | 200MB |
| `MEDIA_RENDER_MAX_CONCURRENT_PER_USER` | 2 |
| `MEDIA_RENDER_JOB_TIMEOUT_SEC` | 900 |
| `MEDIA_RENDER_EPHEMERAL_RETENTION_DAYS` | 7 |
