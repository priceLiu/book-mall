# CineCanvas影视AI全流程协同平台

## 完整功能设计文档（V2.5 终版定稿）


### 文档说明

1. 本文档为**最终唯一落地版本**，所有逻辑经过逐轮对齐，可直接交付开发；
2. 核心原则升级为**“流程有引导，但不强制”**——系统提供标准化最佳实践，但尊重团队真实工作习惯，不设死锁；
3. **流程可逆**：剧本、集数、角色/场景/道具均支持增删改，修改后系统自动检测影响范围并提示，不阻塞作业；
4. **全局资产不锁定**：系统引导“建议先做全局资产”，但不强制，各集生产在项目创建后即可开启，专业团队自行安排节奏；
5. 文档包含：顶部控制台交互详解、全局资产制作流程、分集生产流程、流程变更处理机制、画布交互、资产体系、编码字段规范、完整测试用例。


### 一、产品核心定位与底层架构（定版）

#### 1.1 产品定位

面向短剧/网剧/中小型影视团队的AI可视化无限画布工业化协同平台。

**核心设计理念**：“流程有引导，但不强制”——系统提供影视工业化的标准化最佳实践，同时尊重团队的真实工作习惯和灵活性。

#### 1.2 最终定型核心架构

**四层固定架构：**

| 层级 | 名称 | 说明 |
| :--- | :--- | :--- |
| 1 | **IP总项目层** | 全局唯一，承载项目配置、全局资产池、总进度管控 |
| 2 | **顶层全局总画布** | 剧本创作 + 全局资产统一制作（角色/场景/道具/风格） |
| 3 | **数据层：项目资产面板** | 存储剧本拆解数据、全局资产、各集资产、变更历史 |
| 4 | **生产层：自由画布** | 成员按需创建，通过顶部控制台驱动生产 |

#### 1.3 核心业务原则

1. **流程有引导，但不强制**：系统显示“推荐先做全局资产”，但不锁定任何功能；
2. **修改是常态**：剧本、集数、角色均可增删改，修改后系统自动检测影响范围，推送通知；
3. **画布完全自由**：成员可在任何时间做任何事，系统不设死锁；
4. **资产统一管理**：所有已发布资产全画布可引用，无需审批；
5. **变更可追溯**：所有修改记录版本、原因、影响范围。


### 二、顶部控制台：全功能生产控制台（V2.5）

#### 2.1 整体布局


**三大区域：**

| 区域 | 名称 | 内容 |
| :--- | :--- | :--- |
| **A区** | 项目标识 | 项目名称 + 返回首页 |
| **B区** | 全局资产区 | 角色/场景/道具/风格 四个模块（始终可用） |
| **C区** | 各集生产区 | 所有集数方块（始终可点击，不锁定） |

#### 2.2 B区：全局资产区（始终可用）

- 四个模块始终可点击，随时制作
- 每个模块显示完成状态：🟢已完成 / 🔵进行中 / ⬜未开始
- 点击后弹出资产清单，选择生成节点

#### 2.3 C区：各集生产区（始终可点击，不锁定）

- 所有集数方块始终可点击，不设锁定
- 但系统会显示**推荐路径提示**：“建议先完成全局角色/场景/道具/风格，再进入各集分镜制作（不强制，可跳过）”
- 点击集数后，画布生成该集分镜脚本→分镜图→分镜视频→音效→对白→成片工作流
- 如果全局资产尚未完成，节点内显示“⚠️ 建议先完成全局资产以确保统一性”，但不阻塞

#### 2.4 推荐路径提示（非强制）

| 位置 | 提示内容 | 显示条件 |
| :--- | :--- | :--- |
| 顶部横幅 | “💡 推荐路径：建议先完成全局角色/场景/道具/风格，再进入各集分镜制作” | 全局资产未全部完成，且用户首次进入C区 |

| 提示可关闭 | 点击“不再提示”后永久关闭 | 用户主动关闭 |


### 三、流程变更与柔性适配机制（核心新增）

#### 3.1 变更类型与系统响应

| 变更类型 | 系统响应 | 影响范围追踪 |
| :--- | :--- | :--- |
| **剧本修改** | 自动检测哪些集/角色/场景受影响 → 标记🟠“需审阅” → 推送通知 | ✅ 自动完成 |
| **增加集数** | 在C区新增集数方块（灰色）→ 剧本追加内容 → 全员可见 | ✅ 自动完成 |
| **删减集数** | C区移除该集方块 → 该集资产标记“已归档” → 通知关联成员 | ✅ 自动完成 |
| **增加角色** | 全局资产B区新增角色条目 → 提示“新角色需制作” → 通知管理员 | ✅ 自动完成 |
| **删减角色** | 全局资产B区角色标记“已废弃” → 通知引用该角色的集数负责人 | ✅ 自动完成 |
| **增加场景/道具** | 同上 | ✅ 自动完成 |

#### 3.2 变更处理流程

**标准流程：**

```
步骤1 : 管理员在剧本节点发起“修订”
步骤2 : 系统自动创建修订草稿
步骤3 : 管理员/编剧修改内容（增删改集数、角色、场景、道具）
步骤4 : 系统自动检测变更范围（哪些集数/节点受影响）
步骤5 : 提交审核 → 审核通过 → 发布变更
步骤6 : 受影响节点标记“🟠需审阅”
步骤7 : 负责人确认是否需要调整 → 调整后重新提交 / 确认无需调整
```

#### 3.3 变更影响检测逻辑

| 变更内容 | 检测规则 | 影响标记 |
| :--- | :--- | :--- |
| 第05集剧情修改 | 检测该集所有节点（分镜/图/视频/音效/对白/成片） | 标记“需审阅” |
| 全局角色删除 | 检测所有集的分镜/分镜图/视频中引用该角色的节点 | 标记“需审阅” |
| 全局场景修改 | 检测所有集的分镜/分镜图中引用该场景的节点 | 标记“需审阅” |
| 新增集数 | 无影响检测 | 仅新增方块 |
| 删除集数 | 检测该集所有资产 | 标记“已归档” |

#### 3.4 变更历史追溯

- 所有变更记录在项目资产面板“变更历史”页签中
- 每次变更记录：谁、何时、变更内容、影响范围、处理状态

### 九、终版总结

1. **核心原则**：流程有引导，但不强制——尊重真实工作习惯；
2. **全局资产不锁定**：各集生产随时可开启，系统仅做推荐提示；
3. **流程可逆**：剧本、集数、角色均可增删改，自动检测影响范围；
4. **变更可追溯**：所有修改记录版本、原因、影响范围和处理状态；
5. **顶部控制台**：B区全局资产 + C区各集生产，全部可用，无死锁；
6. **推荐路径**：以提示横幅呈现“建议先做全局资产”，非强制，可关闭。
