# 定价与风控（财务 2.0 现网口径）

> **权威数字**以 [`seed-credit-billing.ts`](../book-mall/lib/billing/seed-credit-billing.ts) 与 [`财务2.0-验收标准.md`](./财务2.0-验收标准.md) §1 为准。下文已对齐现网 seed，非早期演算稿。

## 前置说明

- 单条漫剧视频最大 **15 秒**；按秒计费，超过仍按 15 秒计
- 视频系数 **M=4**，毛利护栏 **≥75%**
- 积分双池：通用池 + 视频专用池（`MembershipPlan.videoMonthlyCredits`）
- 计价真值：`ModelCostProfile` → `ModelCreditPrice` → Gateway 扣积分（**无** ToolBillablePrice 并存）

## 二、视频成本（核算模型 happyhorse-r2v）

- 挂牌 0.9 元/秒，渠道折扣 10% → **净成本 0.81 元/秒**
- 15 秒成本 = 0.81 × 15 = **12.15 元**
- M=4 → 挂牌价 = **48.6 元/条**
- 毛利 = 1 − 12.15/48.6 = **75%**

### 个人月付单条扣分（高级版示例）

|------|-----|
| 单价 | 0.030615 元/积分 |
| 扣分 | round(48.6÷0.030615) = **1587** |
| 实收 | 1587×0.030615 ≈ ¥48.59 |

完整五档见验收标准 §1.1。

| 任务 | 个人含/月 | 团队含/席/月 | 超额扣分 |
|------|-----------|--------------|----------|

**无套餐含次（仅统计）**：纯文字对话（`CHAT` 无视频附件）、`OTHER` 及无法映射的调用。

### 7.1 计费类别与 Gateway 映射（Book Single Writer）

Gateway 只记调用日志；**套餐扣次 / 超额扣分 / 财务明细类别** 由 Book `mapLogToByokTaskKind` 统一映射（实现：`book-mall/lib/billing/byok-pricing.ts`）。

| Book 类别 | 判定（优先级自上而下） | BYOK 套餐含次 |
|-----------|------------------------|---------------|
| 文生图 | `requestKind=IMAGE`；`requestKind=TRYON`（含 `aitryon*`、`aitryon-parsing-v1`） | 有 |
| 图生视频 | `requestKind=VIDEO` 且无 v2v 参考视频信号 | 有 |
| 视频生视频 | `requestKind=VIDEO` 且 `inputSummary` 含 `sourceVideo` / `videoUrl` / `referenceVideo` / `mode=v2v` 等 | 有 |
| 视频理解 | `requestKind=CHAT` 且 `inputSummary.input.messages` 含 `video_url` 部件 | 有 |
| TTS / 语音 | `requestKind=TTS` | 有 |
| 文字 | `requestKind=CHAT` 无视频附件 | 无（仅统计） |
| 其他 | `OTHER` 或无法映射 | 无 |

明细列始终展示 **模型**（`canonicalModelKey` / `model`）；试衣类另写 `tryonModelKey`。

### 7.2 平台代付与七类关系

平台代付（`billingPersona=PLATFORM_CREDIT`）**权益形态**与 BYOK 不同：

| 维度 | 平台代付 | BYOK（§七） |
|------|----------|-------------|
| 权益 | §一 月积分双池（通用 + 视频） | 月费 + 五类含次 + 超额轻量包 |
| 扣费 | 按 `ModelCreditPrice` 逐模型扣分 | 套餐内扣次；超额固定积分 |
| 个人中心 | 七类「成功/失败/扣积分」统计 | 五类含次 + 剩余 + 文字/其他 |
| 报表 `任务类型` | 七类标签（与 BYOK 相同 taxonomy） | 同上 |

**不变**：§一 月积分总量、视频池比例不因七类拆分而调整。

**扣费归类**：结算时 Book 写入 `billingCategory`（实现：`classifyBillingCategory`）；`TRYON` 归入「文生图（含试衣）」；纯 `CHAT` 为「文字」；`OTHER` 为「其他」。

**价目**：平台路径须有 `ModelCostProfile`（试衣 `aitryon*`、VL `qwen3-vl-*`、TTS `qwen3-tts-flash` 等）；见 `scripts/seed-platform-model-costs.ts`。

## 八、工具技术服务费（Phase D，与 §一 会员套餐并列）

> **财务 2.0 核心文档（§一～§七）不含本表**；这是工具站 **按 navKey 的准入月费**，与「会员套餐积分」「BYOK 套餐额度」是第三条收费线。

### 8.1 怎么收

| 环节 | 实现 |
|------|------|
| 定价表 | `ToolServiceFeePlan.monthlyFeePoints`（后台 `/admin/tool-service-fee` 可改） |
| 开通/续订 | 账户「工具技术服务费」→ `activateToolServiceFee` |
| 扣款账户 | **通用积分池** `consumeCredits`（`charge-monthly.ts`；非旧钱包、非按次 usage） |
| 周期 | 30 天；到期须主动续订（无自动续费 cron） |
| 准入 | `assertPlatformGatewayEntitlement`：无有效 `UserToolServicePeriod` → 403 |

### 8.3 与 §七 BYOK、§一 会员的关系（避免双扣误解）

```
计费身份 billingPersona（注册时选定，互斥锁定）
  PLATFORM_CREDIT → 会员套餐（§一）积分池 + 自动隐藏 sk-gw；前台无 Gateway Key 绑定 UI
  BYOK            → §七 月费 + 任务额度 + 用户自建 sk-gw；超额扣 §一 轻量包

会员套餐（§一）     → 充值得月度积分池（通用+视频），团队代付时按模型扣
BYOK 月费（§七）    → 自备 Key 用户的平台技术服务 + 月度任务额度 + 超额轻量包
工具技术服务费（§八）→ 每个 navKey 的「进入工具」月费；BYOK 与平台代付均须有效 navKey 月费
legacy 课程订阅     → 仅学堂内容，不再解锁 AI 工具
```

**产品收敛建议（待拍板）**：长期应合并 §七 与 §八，避免「BYOK ¥39 + 文生图 ¥20 + 图生视频 ¥30」叠收。过渡口径：

- **月费（§八）** = 工具使用权（能点开、能调 Gateway）
- **BYOK 额度（§七）** = 平台编排次数；超额才扣 §一 通用积分
- **厂商费** = 用户 Gateway Key，Book 不代收

单次生成 **不再** 走旧 `ToolUsageEvent` 钱包按次；明细以 `GatewayRequestLog` + `CreditLedger` 为准。
