# 财务 2.0 — 测试用例

> 围绕**视频生成 / 个人 / 团队**，覆盖积分换算、冻结扣费、退款、视频池隔离、团队分账、对账明细与差异、毛利护栏。
> 全部用**库中阿里挂牌价** happyhorse-r2v（净 0.81 元/秒、15s、**M=1.0 贵视频**）核算。预期数字见 `财务2.0-验收标准.md` §1。
>
> 落地：纯函数层用 `vitest`（`pnpm --filter book-mall test`，无需 DB）；集成层为带 DB 的脚本/手测步骤。

## A. 纯函数单测（vitest，无 DB）

文件：`book-mall/test/unit/credit-pricing-formulas.test.ts`、`book-mall/lib/billing/__tests__/reconciliation-diff.test.ts`

### TC-A1 逐档积分换算（核心）
输入 `listPriceYuan=12.15`（happyhorse 15s · M=1.0），对每档单价断言 `computeTierCredits`：

| 单价 | 期望积分 |
|------|----------|
| 0.069000 | 176 |
| 0.049667 | 245 |
| 0.046000 | 264 |
| 0.042786 | 284 |
| 0.039967 | 304 |
| 0.036006（团队高级） | 337 |
| 0.032990（团队豪华） | 368 |
| 0.030015（团队至尊） | 405 |

### TC-A2 贵视频各档毛利 ≈ 0%
对每档 `computeEffectiveMargin({netCostYuan:12.15, creditsPerUnit:<上表>, pricePerCreditYuan:<单价>})` 的 |margin| ≤ 0.01。

### TC-A3 视频 15s 封顶
`videoBillableSeconds(20) === 15`；`videoBillableSeconds(8) === 8`（<15 据实，本期固定 15s 业务侧恒传 15）。

### TC-A4 分类护栏
- `marginGuardFor("PER_SEC") === -0.02`；
- `assertMarginOrThrow(margin=-0.05, guard=-0.02)` 抛 `MarginGuardError`；
- `assertMarginOrThrow(margin=0, guard=-0.02)` 通过。

### TC-A5 对账差异纯函数
`diffReconciliation(internal, vendor, tolerance=0.05)`：
- internal=`{happyhorse:121.5}`、vendor=`{happyhorse:121.5}` → `status=OK`、`diffYuan=0`。
- vendor=`{happyhorse:130.0}` → `diffYuan=8.5`、`status=OVER`。
- vendor 缺该项 → `status=MISSING_VENDOR`。

## B. 集成场景（带 DB）

前置：`pnpm --filter book-mall db:deploy && db:seed`，确保 happyhorse-r2v 成本档与个人/团队套餐已落库；视频模型按 `model-margin-policy` 已发布。

### TC-B1 个人视频生成 + 冻结扣减（高级版）
1. 用户 U 高级版，视频池余额 6500。
2. 发起 1 条 happyhorse 15s → `reserveCredits(264)` → 视频池冻结，余额冻结后可用 = 6500−264=**6236**。
3. 渲染 SUCCEEDED → settle → 写 `creditsCharged=264`、`costSnapshotYuan=12.15`、`marginSnapshot≈0`。
4. 厂商全失败 → 全额退还 264 积分。

### TC-B2 视频池与通用池隔离
视频扣减只减 `videoBalanceCredits`；文本/图像扣减只减通用池。

### TC-B3 团队 4 席高级版
团队豪华版 happyhorse 15s 扣分 **369**；Seedance 720P 15s 扣分 **405**（对标竞品积分）。

### TC-B4 Scenario Lab
- 运行 `test/unit/scenario-lab.test.ts`：个人高级版 30 行毛利符合 M 分档（容差 2.1pct）。
- 团队 4 席高级版：同上。

### TC-B5 调价六维测算
- 运行 `test/unit/pricing-simulation.test.ts`：M=1.0 贵视频全档护栏通过；反向验算目标 33% → M≈1.5。
