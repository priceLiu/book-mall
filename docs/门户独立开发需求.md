# 门户独立开发需求

> 目标：让「画布 / 快速复制 / 电商工具箱 / 故事版」各自成为可独立推广、独立 SEO 的门户，用户可直接在各门户域名注册/登录，界面按品牌换皮；用户与积分全站互通、无感切换；后台、财务、账本、准入策略不变。工具站仍由 Book 进入。
>
> 关联文档：`book-mall/doc/product/12-platform-app-federation.md`、`book-mall/doc/plans/2026-platform-unification-rollout.md`
> Cursor 规则：`.cursor/rules/platform-app-federation.mdc`、`.cursor/rules/site-architecture-doc.mdc`

## 1. 背景与现状

平台已是联邦架构，Book 为唯一身份源（NextAuth + 用户表 + 注册/登录页），子应用经 SSO 换票获取会话：

- 部署独立：✅ 每应用独立进程/域名/服务。
- 会话独立：✅ 每应用自域 `tools_token` cookie（不共享 Book 的 NextAuth cookie）。
- 入口独立：❌ 登录/注册/个人中心 UI 仅在 Book 域；用户首次进门户被弹到 book 登录。

换票流程（现状）：子应用无 `tools_token` → `RequireAuth` 跳 Book `/api/sso/tools/re-enter?app=xxx` → 未登录再跳 Book `/login` → 登录后签发一次性 code → 回子应用 `/auth/sso/callback` → 换 `tools_token`。

准入闸门（现状问题）：`issueToolsSsoRedirect` / `/api/sso/tools/exchange` / `issueToolsAccessTokenForUser` 都要过 `getToolsSsoEligibility`（管理员或有效工具月费）或电商准入，否则 403；`introspect` 未开通返回 `active:false`。因此新注册无月费用户拿不到会话、进不了门户。

## 2. 目标态

- 各门户在自身域名承载品牌化 登录/注册/个人中心 UI。
- 认证仍走 Book：复用凭证校验、短信 OTP、注册逻辑；不重写认证逻辑（只换皮）。
- 换票机制保留不变（OIDC 风格 authorization code），仅解耦「签发 token」与「工具准入」。
- 用户与积分全站互通、无感切换：Book 为唯一 IdP + 唯一积分账本。
- 各门户独立 SEO（域名、sitemap、robots、metadata、SSR 落地页）。
- 后台、财务、账本、准入策略：单点写入仍在 Book，不变。

## 3. 已定产品决策

注册/登录成功即可进入门户（可浏览首页、定价、个人中心、开通引导）；「生成 / 工具能力」在调用时由 `assertPlatformGatewayEntitlement` 按工具月费 + Gateway 关联拦截，未开通则提示开通。理由：最大化注册转化，且与「gateway 每次复查 entitlement」的安全模型一致。

## 4. 认证复用原则（不要多套逻辑）

- 抽取共用函数 `verifyCredentialsLogin()`（`book-mall/lib/auth/verify-credentials.ts`），NextAuth `authorize()` 与门户端点共用（唯一真源）。
- 注册直接复用 `POST /api/auth/register`；短信复用 `lib/auth/sms-verification-service.ts`。
- 子应用零认证逻辑，仅品牌化表单 + 薄 BFF 转发 + 一个 Book 桥接页。→ 一套逻辑，多套皮肤。

## 5. 目标登录 / 无感切换流程

```text
用户 -> 门户/login(品牌页,可SEO) -> 门户 BFF
门户 BFF -> Book /api/sso/portal/verify 或 /api/auth/register (server secret)
Book -> 校验通过 -> 返回 autoLoginToken
门户前端 -> 整页跳 Book /portal-signin?t=..&app=..&redirect=..
Book /portal-signin -> NextAuth signIn(autologin) 建立 Book 会话(共享身份)
Book -> /api/sso/tools/re-enter 签发一次性 code -> 302 门户 /auth/sso/callback
门户 /auth/sso/callback -> exchange -> tools_token(member/gold/admin)
之后跨门户: 已具 Book 会话, 静默 re-enter = 一次不可见重定向(无需再输入)
```

积分因单账本自然一致；`introspect` 返回 `credit_balance` / `credit_pools`，各门户个人中心直接展示。

## 6. Book 侧改动

| 项 | 说明 |
|---|---|
| 共用校验 | 新建 `lib/auth/verify-credentials.ts`：`verifyCredentialsLogin()`；`lib/auth.ts` authorize 调它。 |
| 门户校验端点 | 新建 `app/api/sso/portal/verify/route.ts`：`Bearer TOOLS_SSO_SERVER_SECRET` 授权；校验后 `issueAutoLoginToken` 返回 `{ autoLoginToken, phone }`；不判准入。 |
| 登录桥接页 | 新建 `app/(site)/portal-signin/page.tsx`：复用 NextAuth `autologin` 建立 Book 会话 → 跳 `re-enter`。 |
| 准入解耦 | `tools-sso-token.ts` 增 `member` tier；`issue-tools-access-token-for-user.ts` / `exchange` / `issue-tools-sso-redirect.ts` 未开通不再 403，签发 `member` 令牌（navKeys 可空）；`introspect` 有效令牌+会话即 `active:true`，新增 `entitled` 字段。 |
| SMS 转发 | `/api/auth/sms/send`、`/api/auth/register` 由子应用 BFF 服务端调用（dev 回传 mockCode）。 |

## 7. 子应用改动（quick-replica / canvas / e-commerce / story 各一份）

> 故事版（story-web，`app=story`，端口 3003）与前三者同一套模式：复用 Book 认证（`/api/sso/portal/verify` + `/api/auth/register` + `/api/auth/sms/send`），仅 BFF 转发 + 品牌换皮（dark `twenty-*` / `--story-*` + `.story-input`）。首页/空间为公开落地页（Book 观众会话），受保护路径 `/projects`、`/project/*` 由客户端 `RequireAuth` 静默 re-enter，耗尽后跳本域 `/login`（不再弹 Book）；`PortalNav` 挂在 `StoryShell` 头部并含「故事版」入口，`StoryAuthBar` 提供 登录/注册/退出。

| 项 | 说明 |
|---|---|
| BFF 转发 | `app/api/auth/login`、`app/api/auth/register`、`app/api/auth/sms/send`：服务端 secret 调 Book，不暴露到浏览器。 |
| 品牌化页 | `app/login`、`app/register`、`app/account`，按各自设计系统换皮（qr-* / EcomButton* / Pro2 紫罗兰）。 |
| RequireAuth | 未认证跳本域 `/login`（不再弹 book）；保留静默 re-enter 处理「已在 Book 登录」。 |
| SEO | 首页/落地/定价 SSR 且不拦登录；`app/sitemap.ts`、`app/robots.ts`、各页 metadata。 |
| 跨门户导航 | 共享 `PortalNav` + origins（`NEXT_PUBLIC_*_ORIGIN`）；工具站指向 Book。 |

## 8. 配置

各子应用补 `.env.example`：`MAIN_SITE_ORIGIN`、`TOOLS_SSO_SERVER_SECRET`、`NEXT_PUBLIC_*_ORIGIN`。

## 9. 安全

- 无头 login/register 经子应用 BFF + 服务端 secret，不暴露 `TOOLS_SSO_SERVER_SECRET` 与短信接口到浏览器。
- `tools_token` 短 TTL；生成前每次复查 entitlement（`assertPlatformGatewayEntitlement`）。
- 财务/账本/准入仍 Book 单点写入，子应用零 DB 写权限。

## 10. 不改动项（边界）

- 换票机制（SSO code + exchange + tools_token）保留。
- Book NextAuth、用户表、注册核心逻辑不重写。
- 财务、账本、订阅、工具月费、Gateway 计费与准入策略不变。
- 工具站入口仍由 Book 进入。

## 11. 无需迁移

`SsoAuthorizationCode` 表已存在，本需求全为逻辑改动，无需数据库迁移。若实现中确需迁移，按防火墙绕过路径 `pnpm db:apply-pending` + `pnpm db:generate` 执行。
