# 工具站计费计费规则说明

本文说明 AI 工具站与 **费用、扣费口径、超时与免责** 相关的 **计划规则与公示**，便于用户理解与合规展示。具体单价以主站定价表及「费用使用明细」实际入账为准。

<!-- 以下为章节标题后缀 `{.warn}` `{.banner}` `{.neutral}` 用于站内页样式映射 -->

## 一、适用范围与免责提示 {.warn}

- 本说明用于 **产品公示与用户知情**，不构成法律承诺；遇政策、第三方 API、定价策略调整，以 **主站 / 工具站届时公示及后台配置** 为准。
- **工具应用消费**与站内 **课程订阅（月度 / 年度等）相互独立**：订阅权益 **不自动等同于** 工具站抵扣额度；工具使用以 **钱包充值与单次标价扣费** 为主。
- 因 **网络中断、浏览器关闭、云服务商故障** 等导致的展示延迟、重复提示或短暂状态不一致，以 **服务端为准**；异常时请优先核对「费用使用明细」或联系支持。

## 二、充值、水位线与准入 {.banner}

- 工具应用统一采用 **充值钱包** 方式扣费；平台可配置 **最低余额水位线**（当前约定默认 **20 元人民币** 等价分值，以主站 `PlatformConfig` 为准）。
- **黄金会员（工具站准入）** 等与课程「高级计量订阅」为 **不同规则**：准入通常要求 **历史有过充值记录** 且 **当前余额不低于水位线**，详见主站文档 `gold-member` 实现。
- 单次调用是否允许继续，以 **实时余额** 与 **`introspect`** / 业务接口返回为准。

## 三、费用使用明细页展示什么 {.neutral}

- 「费用使用明细」列表与主站 **`ToolUsageEvent`** 对齐：**仅展示已成功入库、且具备计费金额的流水**。
- **页面浏览、未标价动作** 一般 **不入库、不计费**，明细中不会出现「纯浏览」条目。
- 分页：**每页最多 50 条**，可通过接口 `page` / `limit` 翻页（与主站 `GET /api/sso/tools/usage` 约定一致）。
- 列表未展示金额的位置可能显示「—」，表示该条 **非扣费流水或未标价**，不等同于扣费为 0 的历史争议，以明细备注与主站后台为准。

## 四、AI 试衣计费口径 {.banner}

- **计费触发**：用户在 **AI 试衣** 流程中，大模型任务 **成功返回成片**（任务成功且具备结果图 URL），且工具站向主站 **成功上报** `try_on` 后，记 **一条** 计费事件。
- **打点**：`toolKey = fitting-room__ai-fit`，`action = try_on`；**不在** 试衣间套装浏览、我的衣柜浏览、费用明细 **纯浏览** 时计价。
- **单价**：与主站 **工具定价表 / 台帐** 一致（文档示例曾为 **1 元 / 次成功试衣**，实际以配置为准）。
- **幂等与轮询**：同一 `taskId` 重复轮询由服务端 **去重**，**不重复扣费**。成片页可展示 **本轮标价金额** 或幂等提示文案。
- **失败 / 进行中**：任务失败或未完成 **不计费**（不产生对应标价流水）。

## 五、文生图计费口径 {.banner}

- **计费触发**：任务在模型侧为 **`SUCCEEDED`**，工具站 **`/api/text-to-image/settle`** 校验成功后，向主站上报 **`invoke`**（`toolKey = text-to-image`），按 **单次生成** 标价扣费（示例口径曾为 **约 0.5 元 / 次**，以定价表为准）。
- **幂等**：同一 **`meta.taskId`**（DashScope 任务 id）重复结算走 **幂等**，**不重复扣款**。
- **余额不足（402）**：可能仍可依链接 **短时查看生成结果**，但计费未完成；需充值后可再次生成或按提示处理。

## 六、超时、前端中断与云厂商费用 {.warn}

- **用户钱包扣费**：以是否 **成功执行并完成 `settle` / 上报** 为准。前端 **轮询超时、关闭页面** 若导致 **未调用结算**，则 **通常不会对用户记该次工具扣费**（除非用户稍后使用「重试计费」且任务已成功）。
- **文生图前端轮询**：约 **每 2 秒** 查询任务状态，最多约 **90 次**，超时后界面提示用户核对明细；单次 HTTP **无单独超时** 时极端情况下可能长时间停留在「生成中」。
- **云厂商（如阿里云 DashScope）**：任务若在云上 **已成功推理**，**有可能仍对持有 API Key 的账号计费**，与浏览器是否等到结果 **无关**；此为 **平台供应商账单**，与用户钱包流水 **分列**。产品目标是用户侧 **按成功结算计费**，云侧成本由运营侧承担并对账。

## 七、计费上报可靠性（工具站 ↔ 主站）{.neutral}

- 工具站对主站 `POST /api/sso/tools/usage` 使用 **带退避的重试**（网络抖动、502–504 等）；文生图结算失败时可 **保留图片** 并提供 **「重试计费」**（同一任务 id 幂等）。
- AI 试衣在计费未确认前，前端可 **继续轮询** 直至终态或超时，并以 **提醒样式** 提示异常；最终以明细入账为准。

## 八、管理后台与定价 {.neutral}

- 主站 **工具应用管理**：维护菜单可见性、各工具 **单价、生效 / 结束时间**（结束时间为空可视为长期）。
- **单价变更** 仅对变更后新产生的调用生效；历史流水不因改价回溯修改。

## 九、修订记录 {.neutral}

- 文档随仓库迭代更新；重大变更建议在产品公告或站内显著位置同步提示。
