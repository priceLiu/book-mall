# 学习端 · 轻量报价体系 — 需求说明

> **定位**：教育 / 学习向工具站，计费与价目展示 **保持轻量**，避免建设完整「价目中台」。  
> **范围**：仅 **中国内地** 部署与对外口径；**不承接**阿里云百炼「免费额度」类对客承诺。  
> **文档性质**：产品需求基线，供评审、备查与 **下一平台复制** 时对照。

---

## 1. 背景与目标

- 工具站需要 **可解释的对外单价**（用户可见、运营可核对），又与上游（如 DashScope / 百炼）成本大致可比。
- `tool-web/doc/price.md` 为官方价目快照，**仅作人工对照**，不作为运行时代码唯一数据源。
- 通过固定 **零售价系数**，降低维护面：优先保证 **已上架模型** 报价准确，而非全行业全模型全覆盖。

## 2. 范围（In Scope）

| 项 | 要求 |
|----|------|
| 地域 | **仅中国内地**（与当前接入地域、数据驻留说明一致；不在学习端产品文案中突出「全球 / 国际」价）。 |
| 对客报价 | 在内部 **成本单价**（与官方中国内地价目一致或可审计来源）基础上，统一 **× 2.0** 作为 **对外零售价**（人民币，保留小数规则见解决方案）。若界面使用 **点数**，须与主站钱包 **同一套分（minor）账本** 对齐，见 `learning-pricing-wallet-points.md`。 |
| 免费额度 | **学习端不对用户宣传** 百炼免费额度；若页面需合规提及第三方，可表述为「以云服务商官网为准」，**不把免费额度作为本产品权益**。 |
| 学习端体量 | 价目以 **子集** 维护：仅覆盖 **本站实际可调用的模型 / 工具**；随工具上架增量追加。 |
| 文档可继承 | 同一套 **需求 + 解决方案** 文档结构可拷贝到新仓库 / 新平台，仅替换地域、系数、模型清单等配置项。 |

## 3. 非目标（Out of Scope）

- 不在学习端首期建设：**全量** 解析 `price.md`、自动爬取官网、多币种、多区域完整价值目表。
- 不承诺与阿里云账单 **逐笔逐符号** 一致（云侧账单为权威）；本产品以 **公示标价 + 实际扣费流水** 为准。
- 不把 **Batch 半价、上下文缓存折扣** 等复杂规则在学习端 **首期** 全自动仿真（可作后续增强，见解决方案「扩展位」）。

## 4. 角色与术语

- **成本单价**：与选定官方价目（中国内地）对齐的基准，内部对账用；**不强制**在 C 端大字展示。
- **零售单价**：成本单价 × **RETAIL_MULTIPLIER**（本项目约定 **2.0**），用于展示、预估与 **主站工具定价配置** 的建议值/绑定值（以最终实现为准）。
- **工具 / toolKey**：与现有 `book-mall` 工具打点、`tool-web` 路由一致；每个可对客计价的入口应对应 **一条或少量** 价目绑定。

## 5. 验收标准（首期）

1. 书面公式：**零售 = 成本 × 2.0** 写入住 `learning-pricing-solution.md`，并在团队内达成一致。  
2. **已上架工具**（清单在解决方案中维护）均能在文档或配置中找到 **成本价来源**（官网章节 / 表格行 / `price.md` 锚点）。  
3. 用户可见文案 **不出现「本产品赠送百炼免费额度」**；若提及第三方定价，指向官网或内部说明页。  
4. 修订可追溯：本文 **§7 修订与跟踪** 表有记录；重大系数或地域变更走修订记录。

## 6. 依赖与假设

- 主站具备或即将具备：**工具单价配置**、钱包扣费、`ToolUsageEvent` 流水（与现有 `billing-plan-rules.md` 一致）。  
- 学习端 API Key 与账单由运营侧持有；**云侧成本** 与用户侧扣费 **分列管理**（见既有计费说明）。
