付费的规则
1. 月度, 年度, 凡是订阅方式的, 只用于课程, 不用于工具应用.
2. 工具应用统一以充值的方式, 才能使用, 最低水位线为 20 元人民币.规则如下:
   a. 每个工具在调用大模型时, 调用一次计数一次, 最终按计数收费.
   b. 每个工具按次数收费, 工具的单价不一样, 调用多次, 就乘以单价, 即为该工具使用费用.

### AI试衣（`fitting-room__ai-fit`）计费口径

- **计费触发**：用户在 AI试衣页点击「试衣」，大模型任务 **成功返回成片**（任务状态成功且具备结果图 URL），且服务端完成一次记录后，记 **一条** 计费事件。
- **打点动作**：主站 `ToolUsageEvent.action = try_on`，`toolKey = fitting-room__ai-fit`；**仅在上述成功后** 由工具站试衣 API 写入（经 SSO 上报主站），**不在页面浏览（`page_view`）时计价**。
- **单价**：与定价表 / 台帐一致，当前约定 **1 元 / 次成功试衣**（每次成功成片对应一条带金额的 `try_on` 流水）。失败、进行中不计费；同一任务重复轮询已由服务端去重，不重复计费。
- **非计费**：试衣间入口页、我的衣柜、费用使用明细等 **浏览** 不产生 AI试衣扣费，且 **不入库** `ToolUsageEvent`。

### 计费上报可靠性（工具站 ↔ 主站）

- **文生图**：`/api/text-to-image/settle` 与工具站内 **`postToolUsageFromServerWithRetries`** 对主站 `POST /api/sso/tools/usage` 做 **多次指数退避重试**（502–504 / 429 / 网络异常）；前端在 **出图后立即展示结果**，再在后台结算；失败时保留图片并提供 **「重试计费」**（同一 `taskId` 幂等）。
- **AI 试衣**：成片 GET 内同样使用带重试的上报；仅在 **`recorded`** 或 **余额不足（402）** 等终态后停止对该 `taskId` 的上报尝试；前端在计费未确认时会 **继续轮询** 同一任务直至超时或终态，并在结果区提示计费异常文案。

**示例**：AI试衣成功试衣 **100 次**（100 条已标价 `try_on`），单价 1 元/次 → **100 元**。文生图按 **`invoke`** 等调用计价（如 0.5 元/次），调用 100 次 → **50 元**。
