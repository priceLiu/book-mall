# 漫剧画布 · 操作手册

设计规范（固定尺寸、按钮、弹层、Word 预览、对比图）：[design.md](./design.md)

## 1. 标准流程（必须遵守）

```text
故事主题 → 创作剧本（仅大纲）→ 用户审阅确认 → 输出工作流 → 用户在各列手动批量生成
```

| 步骤 | 节点 | 用户操作 | 系统行为 |
|------|------|----------|----------|
| 1 | **故事主题** | 填写主题、选 LLM，点 **创作剧本** | 创建/连接 **故事大纲**，**仅 LLM 生成大纲**（开场/冲突/高潮/收束） |
| 2 | **故事大纲** | 悬停白纸 → 打开审阅，确认文案无误 | 可编辑/保存；**此阶段不输出媒体列** |
| 3 | **故事大纲** | 点 **输出工作流** | 创建 **角色列 · 分镜列 · 分镜视频列 · 剪映导出** 并连线；从大纲 **拆分** 角色/分镜脚本到各列行（**不自动生成** 三视图 / 分镜图 / 视频） |
| 4 | 媒体列 | 点各列底栏 **全部生成三视图 / 批量生成分镜图 / 生成视频**，或单镜重跑 | 各列独立生成与预览 |

### 1.3 输出工作流 · 数据从哪来（拆分规则）

| 目标列 | 数据来源（按优先级） | 说明 |
|--------|----------------------|------|
| **角色列** | ① 大纲 Tab「角色设定」独立字段 → ② 大纲内嵌「角色设定/人物表」段（表或列表）→ ③ 分镜对白里的说话人（「角色名：」） | **仅大纲** 时若无角色段，会尝试从 **分镜** 推断角色名（外观待补） |
| **分镜列** | ① 分镜 Tab 独立字段 → ② 大纲内嵌「分镜脚本」GFM 表 | 与 hub 预览 Tab 回落逻辑一致 |
| **视频列** | 由分镜列行 **同步** 脚本/对白/引用 | 不单独从 LLM 拉取 |
| **剪映** | 绑定本套 `hubNodeId`；只连 **本套** 分镜/视频列 | 重排时 **底对齐** 三列（280 高贴列底）；禁止跨工作流连线 |

**推荐路径（内容最全）**

```text
创作剧本（大纲）→ 审阅弹层分别生成/保存「角色设定」「分镜脚本」→ 输出工作流 → 各列手动批量生成媒体
```

**最短路径（仅大纲）**

```text
创作剧本（大纲，含内嵌分镜表）→ 输出工作流 → 分镜列有行；角色列可能仅对白推断名 → 建议在审阅里补全角色表后再输出
```

### 1.4 多工作流画布

- 每个 **故事主题 + 故事大纲** 为一套独立工作流，通过 `workspaceIds.scriptHubId` 与列节点 `hubNodeId` 绑定。
- **禁止**用画布上「第一个」故事主题的状态判断其他大纲是否已输出；仅当 **本大纲** 已创建三列时，按钮才显示 **已输出工作流** 并禁用。
- 用户 **手动删除** 角色/分镜/视频/剪映节点后，系统会自动清理 starter 上 stale 的 `workspaceIds`，**可再次** 点「输出工作流」重建。

### 1.5 「输出工作流」前置条件

- 已有 **故事大纲** 正文（`hubCanOutputWorkflow`）。
- 大纲 LLM **不在运行**。
- **本套** 尚未输出过媒体列。

## 2. 审阅与修改

1. 在 **故事大纲** 节点切换段 chip：**大纲 / 角色 / 分镜 / 对白**（角色/分镜可从大纲嵌入段回落展示）。
2. 点击白纸或放大镜 → **审阅弹层**（左编辑 · 右原稿）。
3. 各段可单独 **生成 / 重新生成** LLM 文案；保存后写入对应字段。
4. 输出工作流前若大纲内嵌了角色/分镜表，保存或输出时会 **自动拆到** `characterMd` / `storyboardMd`。

## 3. 媒体列

| 列 | 节点类型 | 默认尺寸 | 批量按钮 |
|----|----------|----------|----------|
| 角色 · 三视图 | `story-character-column` | 560 × 2100 | 全部生成三视图 |
| 分镜图 | `story-frame-column` | 1080 × 2100 | 批量生成分镜图 |
| 分镜视频 | `story-video-column` | 500 × 2100 | 各镜点击生成 / 列底栏批量 |
| 剪映导出 | `jianying-export` | 400 × 280 | 导出 ZIP（绑定本套 `hubNodeId`） |

剪映导出节点在输出工作流时 **为本套工作流新建** 并连线（不复用其它工作流的剪映节点）。

详见 [story-eng.md](./story-eng.md) · [design.md](./design.md)
