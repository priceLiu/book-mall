# 钱包：最低线、扣费顺序与退款结算（实现契约）

> 产品条文见 [../product/02-users-billing-and-balance.md](../product/02-users-billing-and-balance.md)、计量见 [../product/03-metering-llm-and-tools.md](../product/03-metering-llm-and-tools.md)。

## 1. 可用余额与硬闸

- **可用余额** = 充值到账累计 − 已确认扣费 − **冻结**（预授权若有）。  
- **发起** 新的高级型/按量任务前：`可用余额 >= 最低余额线`（默认 20，配置项）。  
- **低于最低线**：拒绝新建任务；进行中任务策略（跑完当前步 / 中断）由实现选型后在本文 **追加一节**。

## 2. 应扣未扣

- 定义：计量已发生或供应商已出账，但 **尚未** 在用户账本记 **最终扣款** 的金额。  
- **退款前** 必须：拉平或锁定待结算 → 从可退金额中 **减去** 应扣未扣（或多退少补在下一周期冲账，须在运营政策中禁止歧义；**默认** 采用 **先扣再退**）。

## 3. 退款算法（概念）

1. `可退候选 = 可用余额`（或用户申请部分退款时的申请额，取 min）。  
2. `应扣未扣 = Σ(未入账消耗)`（含争议预留则另减）。  
3. `实际可退 = max(0, 可退候选 - 应扣未扣)`。  
4. 账户清零或部分扣减后生成 **退款订单**，要写 **审计日志**。

## 4. 幂等与对账

- 每次扣费消费须带 **业务幂等键**（如 `trace_id`）；重复回调仅入账一次。  
- 日对账任务状态写入运维可观测处；差异进 `doc/database/` 或工单。
