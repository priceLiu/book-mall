# v2.0 里程碑：课程站占位 · 订阅中心 · 工具套件权益

> 状态：**进行中**（2026-05-12 起稿）  
> 范围：**book-mall**（主站）与 **tool-web**（独立工具站）

---

## 1. 需求摘要

1. **课程站**：落在主站，不与工具站拆仓库；先提供路由与占位内容，后续接会员 / 课时权益。
2. **订阅中心**：个人中心内集中展示「会员订阅状态」与「当前订阅所含 AI 工具套件」。
3. **工具站准入**：在既有黄金会员 + SSO 模型上，**普通用户须同时具备有效订阅**；JWT / introspect 下发 `tools_nav_keys`，工具站侧栏与页面路由按分组门禁。
4. **套餐细粒度（首期）**：`SubscriptionPlan.toolsNavAllowlist` — **空数组 = 订阅期内可使用套件内全部分组**；非空则仅放行列出的 navKey。

---

## 2. 实现要点

### 2.1 数据模型（book-mall）

- 迁移：`prisma/migrations/20260512120000_subscription_plan_tools_nav_allowlist/`
- 字段：`SubscriptionPlan.toolsNavAllowlist String[] @default([])`

本地 / CI 部署后须执行：`pnpm run db:deploy`（或 `prisma migrate deploy`）。

### 2.2 权益解析

- `lib/tool-suite-nav-keys.ts`：套件权威 navKey 列表（与 tool-web `config/nav-tools` 分组对齐）。
- `lib/tool-subscription-entitlements.ts`：`resolveToolsNavKeysForUser(userId)` 读取当前有效订阅计划。

### 2.3 SSO / JWT（book-mall）

- `getToolsSsoEligibility`：管理员仍可直通；**普通用户 = 黄金会员 ∧ 有效订阅**（`TOOLS_SSO_RELAX_MEMBERSHIP=1` 时开发放行仍可绕过）。
- `resolveToolsNavKeysForUser`：开发环境下若启用 **RELAX** 且无订阅，临时返回整套 keys，避免 exchange 因空白名单拒绝（仅本地）。
- `POST /api/sso/tools/exchange`：JWT 载荷写入 `tools_nav_keys`（经裁剪）。
- `GET /api/sso/tools/introspect`：返回 `tools_nav_keys`、`has_active_subscription`。

### 2.4 工具站（tool-web）

- `lib/tools-jwt.ts`：解析 `tools_nav_keys`。
- `lib/tools-introspect.ts`：**会员**若 JWT 缺 keys（旧令牌），回落 HTTP introspect。
- `lib/apply-tool-nav-entitlements.ts` + `ToolShellClient`：侧栏按权益过滤；`ToolRouteEntitlementGate` 拦截无权的套件路径（`/fitting-room`、`/text-to-image` 等）。

### 2.5 主站前台

- 路由：`/courses`（列表）、`/courses/[slug]`（介绍）、`/courses/[slug]/learn`（**播放器占位**：课时正文 + 完成标记）；数据来自 **Product**（`kind: KNOWLEDGE`、`status: PUBLISHED`），课时结构存 `courseContent` JSON（见 `lib/course-content-json.ts`）。
- `CourseLessonProgress`：记录用户按「课节序号」完成状态。
- `lib/assert-course-learn-access.ts`：**登录 +（管理员或有效订阅）** 方可进入学习页（后续可细化到按课程 SKU）。
- 路由：`/account/subscription` — 订阅中心。
- 导航：顶栏「产品」→「AI 学堂」入口。
- **种子 / 产品**：`pnpm db:seed` 写入示例 **工具型** 三条（AI试衣、文生图、智能客服）及两门知识型课程，后台「产品管理」可见。
- 个人中心：订阅卡片「订阅中心」等（延续）。

### 2.6 工具站 API 硬门禁（tool-web）

- `lib/require-tools-api-access.ts`：`requireToolSuiteNavAccess(navKey)`、`requireActiveToolsSession()`。
- 已对 **试衣 / 文生图 / 智能客服对话 / 图片代理** 等相关 `app/api/**` 路由在处理器入口校验 Cookie 会话与 `tools_nav_keys`（管理员直通）。
- `tool-wallet`、`tool-usage`：要求有效工具站会话（不细分套件）。

### 2.7 后台

- **`/admin/billing`**：订阅套餐除价格外，可配置「工具站套件范围」（全部 / 自定义勾选），写入 `SubscriptionPlan.toolsNavAllowlist`。
- **`/admin/products`**：文案说明课程入口与 `/courses` 关系；「预览 AI 学堂」按钮。
