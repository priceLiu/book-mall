# 概述与名词（〇）

> 技术实施与环境变量见 [../tech/stack-and-environment.md](../tech/stack-and-environment.md)  
> 开发流程见 [../process/development-constraints.md](../process/development-constraints.md)

## 文档目的

明确 **知识工具订阅型网站（AI Mall）** 的产品分类、用户层级、付费与余额规则、前台用户中心、后台能力及业务流程；条文以 **可执行的产品规则** 为主，**不含具体代码**。实现时须与拆分后的各专题交叉对照。

## 核心产品定位

平台聚焦 **知识型产品、工具型产品**，采用 **周期性订阅付费** 与 **账户充值按量消耗** 两轨并行、互不抵扣：订阅对应 **普通型** 权益；**高级型** 与 **大模型等服务** 在订阅有效前提下使用 **充值余额** 按本体系扣费。  
目标：权限清晰、计费可审计、**余额与订阅边界固定**、运营可配置单价与水位参数。

## 核心名词

- **知识型产品**：图文/视频课程、学习资料、教程、专栏等。
- **工具型产品**：在线工具、专业/效率工具、AI 应用等。
- **普通型产品**：订阅内的基础内容或基础工具能力，**不额外扣余额**（与「免单品余额费」同义，非「未订阅免费」）。
- **高级型产品**：进阶内容、高阶工具等；使用前须满足 **订阅有效** 且 **可用余额 ≥ 最低余额线**（默认 20 元，可后台配置），具体按 [02-users-billing-and-balance.md](./02-users-billing-and-balance.md)。
- **订阅付费**：月/年固定周期；**余额不可抵扣订阅费**。
- **充值余额**：用户预付资金，用于高级型/按量服务；**可依规申请提现**，提现前先 **扣缴应结未结费用**（见 02）。
- **游客**：未注册或未登录，或文档 [01](./01-catalog-and-permissions.md) 中约定的浏览身份。
- **免费注册用户**：**产品侧权限与游客一致**（试看、试演示等），区别在于 **拥有个人中心**（账户资料、空订单、消息设置等）；不订阅则不能使用完整学习与工具执行能力。
- **普通订阅用户**：订阅在有效期内，可完整使用全部 **普通型** 知识/工具。
- **高级会员**：**无「充值满固定高额才升级」一类门槛**；在 **订阅有效** 且 **可用余额 ≥ 最低余额线** 时，可使用 **高级型** 知识/工具及大模型等按量能力；**按实际消耗扣费**；余额低于最低线则 **立即停止** 高阶/按量能力（基础订阅权益不变）。详见 02。
