# 用户层级、订阅与余额（二）

## 1. 用户层级与升降级

### 1.1 游客

- **准入**：未注册或未登录。  
- **与免费注册用户对内容权限的关系**：免费注册用户未订阅时，在 **知识/工具使用权限** 上与游客 **对齐**（见一册表格）。

### 1.2 免费注册用户

- **准入**：完成注册、未购买有效订阅。  
- **权限**：与游客 **相同的产品权限** + **个人中心**（见四册）。  
- **升级路径**：购买订阅 → **普通订阅用户**。

### 1.3 普通订阅用户

- **准入**：月/年订阅在有效期内。  
- **能力**：全部 **普通型** 知识/工具。  
- **升级/扩展**：当 **可用余额 ≥ 最低余额线**（默认 **20 元人民币**，后台可配置）时，可使用 **高级型** 与按量服务，系统可将其 **标记为高级会员状态**；**不存在**「必须一次性充值满某大额才能成为高级会员」的固定门槛。  
- **降级**：订阅到期未续费 → **游客**（若仍注册则为「免费注册用户」语义下的未订阅态）。

### 1.4 高级会员（状态）

- **含义**：订阅有效 **且** **可用余额 ≥ 最低余额线**，允许使用 **高级型** 与按量服务。  
- **核心原则**：**按消耗扣费**；**余额低于最低线 → 立即停止使用** 高级型/大模型等依赖余额的能力（**基础订阅内容** 仍按普通订阅规则可用）。  
- **降级**：① 订阅到期；或 ② **可用余额 &lt; 最低余额线** → 失去高级会员状态，回到「仅普通订阅」或「游客/免费注册」视订阅是否有效而定。

### 1.5 「最低余额线」（默认 20 元）

- **定义**：允许 **发起/继续** 使用 **高级型与按量计费服务** 前，账户 **可用余额** 必须 **≥ 该值**。  
- **用途**：避免用户将余额用至无法支付已发生或即将发生的计量费用，保护平台与用户 **应扣未扣** 纠纷。  
- **配置**：金额由 **后台计费配置** 维护（默认 20，单位与币种与充值一致，默认人民币）。  
- **与「阈值」的关系**：本文档所述 **「阈值」即此最低余额线**；**不设** 「充值满 X 元才解锁高级会员」的独立大额阈值。

## 2. 双轨付费体系

### 2.1 周期性订阅（基础权限）

- 月/年套餐，固定价；支持手动续费、自动续费、套餐升级。  
- **订阅费不可用余额抵扣**；订阅到期则普通型权益立即按规则失效。

### 2.2 账户充值与消耗（高阶与按量）

- 充值：固定档位 + 自定义金额；到账记入 **可用余额**（点数，**100 点 = 1 元**）。营销活动「充 N 送 M 点」须用户在 **个人中心 → 充值优惠券**（`/account/recharge-promos`）**领取** 后，在支付/收银环节 **核销**；入账拆分、`Order.meta.topup`（含 `rechargeCouponId`）、流水与对账约定见 **[points-wallet-topup-spec.md](./points-wallet-topup-spec.md)**。  
- **消耗场景**（示例）：高级型工具按次/按时长计费、大模型 Token/调用计费（详见 [03-metering-llm-and-tools.md](./03-metering-llm-and-tools.md)）。  
- **余额有效期**：默认 **长期有效、仅限本平台消耗**；若法规或运营政策变更，以公示为准。  
- **停止使用**：除订阅到期外，**可用余额 &lt; 最低余额线** 时 **禁止使用** 依赖余额的高级型/按量能力，并 **强提示充值**。

## 3. 余额提现与结算规则

### 3.1 可提现原则

- 用户可申请 **余额提现**（或账户注销时的余额处理），平台 **允许** 在符合风控与审计要求的前提下办理。  
- **必须先完成费用结算**：从可申请金额中 **优先扣减「应扣未扣」**（已使用但未入账、延后结算的大模型/工具用量、争议期内预留金额等），再对 **剩余可提现余额** 办理提现。  
- **应扣未扣**：指已实际发生服务或已产生计量事实、依规则应计入用户账单但尚未完成扣款入账的金额；结算完成前 **不可** 对部分金额做「已提现即免责」处理。

### 3.2 与订阅费的关系

- **订阅订单** 的费用处理（是否允许、条件与时限）**单独政策**，与余额提现 **流程独立**；须在公示与订单条款中写明。

### 3.3 公示要求

- 提现路径、处理时效、**应扣未扣** 核算方式、手续费（若有）须在 **会员中心/帮助中心** 公示。

## 4. 逻辑闭环（摘要）

- **订阅** 与 **余额** 边界清晰：余额 **不抵** 订阅价。  
- **高级型/按量** 依赖：订阅有效 + 余额 ≥ 最低线 + 计量规则（三册）。  
- **余额可提现**，但 **先结清应扣未扣**。
