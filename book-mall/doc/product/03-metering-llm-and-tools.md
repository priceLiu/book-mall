# 大模型与高阶工具计量与水位预警（三）

本文档为 **现行建议方案**：上线前由产品在后台配置具体单价与阈值；实施时可技术迭代，但 **水位预警与后台可观测性** 为硬性要求。

## 1. 计量模型（建议）

### 1.1 大模型（LLM）

| 维度 | 建议 | 说明 |
|------|------|------|
| 主计量 | **Token** | 分别配置 **输入单价**、**输出单价**（每千 Token 或每百万 Token，后台可改）。 |
| 辅助 | **调用次数** | 可对低价模型设「每次调用最低消费」防刷。 |
| 扣费时机 | **后置结算为主** | 调用完成后根据供应商返回或自建网关统计的 usage 入账；若需 **预授权**，可冻结与「预估上限」等额余额，结束后 **按实耗结算（多补少扣 / 释放差额）**。 |
| 封顶 | **单次请求预算** | 用户或系统设置 `max_tokens`/金额上限，超限拒绝或截断。 |

**实施要点**：网关层统一采集 `model_id`、tokens、耗时、trace_id；**异步对账**，写入 **消耗明细** 表；失败重试与 **幂等键** 防重复扣款（逻辑细节见 `doc/logic/` 后续成文）。

### 1.2 高阶工具（非 LLM）

| 模式 | 适用 | 计量 |
|------|------|------|
| A. 按次 | API 类、脚本任务 | 每次调用固定扣费或按返回值档位扣费。 |
| B. 按时长 | 音视频处理、长任务 | 按秒/分钟 + 可选起步价。 |
| C. 资源包 | 导出次数、存储配额 | 购买包内扣次，包外用余额按次或引导续包。 |

后台为 **每个工具** 配置：类型、单价、免费额度（若有）、与 **最低余额线** 的联动（见二册）。

## 2. 最低水位线（与二册一致）

- **硬闸**：可用余额 **&lt; 最低余额线（默认 20 元）** → **禁止** 新建大模型/高阶按量任务；进行中的任务按规则 **允许完成当前轮** 或 **优雅终止**（产品需二选一并在逻辑文档写明）。  
- **币种**：与充值一致，默认 CNY。

## 3. 前台水位预警（用户体验）

在 **用户中心 / 钱包 / 工具页** 展示：

| 级别 | 条件（示例，后台可配置） | 行为 |
|------|--------------------------|------|
| 信息 | 余额 &lt; **较高预警线**（如 50 元） | 横幅或气泡：「余额偏低，建议充值以免影响高阶功能」 |
| 提醒 | 余额 &lt; **中等预警线**（如 30 元） | 强提示 + 充值 CTA |
| 严重 | 余额 ≤ **最低余额线**（20） | 拦截高阶入口 + 弹窗 + 充值引导 |

可选：用户 **自定义** 在有高级权益时接收 **短信/邮件/站内信**（开关在四册「消息设置」）。

## 4. 管理后台预警（**重点**）

下列能力为 **V1 即应具备** 的产品功能（可先做简版报表，再增强）。

### 4.1 全站水位看板

- **当前余额总和**、**冻结中金额**（预授权）、**昨日/今日消耗总额**。  
- **低于最低余额线用户数**、**低于各级预警线用户数**（曲线或数字卡片）。

### 4.2 用户级预警清单

- 筛选：**可用余额 &lt; 最低线**、**&lt; 自定义阈值**、**订阅有效但即将无法用高级型**。  
- 列：**用户 ID**、当前余额、24h 消耗、待结算 **应扣未扣** 估值、最近告警时间。  
- 操作：查看明细、发送运营通知（若已实现）、**限制新任务**（风控）。

### 4.3 应扣未扣与对账

- 列表：**延后结算** 批次、金额、关联 trace、**超过 T 小时未入账** 标红。  
- **日对账任务**状态：成功/失败/差异条数；差异进入工单。

### 4.4 异常消耗告警

- 单用户 **单位时间消耗** 超阈值（如 3 倍于近 7 日均值）。  
- 单 **API Key / IP** 异常调用频率。  
- 告警渠道：后台 **告警中心** + 可选邮件/Webhook（实施阶段配置）。

### 4.5 配置项（后台「计费配置」扩展）

- 最低余额线、前台二级/三级预警线。  
- LLM 输入/输出单价、各工具单价。  
- 异常消耗阈值倍数、对账超时小时数。

## 5. 实施顺序建议（研发）

1. 统一 **余额账本** + **扣费流水** + **幂等**。  
2. 接入 **计量采集**（LLM callback / 工具后置计费）。  
3. 实现 **最低线校验** 与 **前台预警**。  
4. 后台 **看板 + 用户清单 + 应扣未扣列表**。  
5. 异常规则与告警。

每一步对应 **数据库变更** 记入 `doc/database/schema-changelog.md`，计费状态机记入 `doc/logic/`。
