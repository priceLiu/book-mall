# 核心业务流程（六）

## 6.1 注册、登录与订阅

1. 浏览 → **注册**（可选）→ 免费注册用户 **仅多个人中心**，产品权限同游客试看边界。  
2. 购买订阅并支付成功 → **普通订阅用户**，解锁全部普通型。  
3. **使用高级型**：**充值**使 **可用余额 ≥ 最低余额线（默认 20 元）** → 系统进入 **高级会员状态**，可按量使用；**无「充一大笔才解锁」的独立门槛**。

## 6.2 产品使用

1. **游客 / 免费注册未订阅**：浏览与试看 → 引导订阅。  
2. **普通订阅**：使用普通型；点高级型 → 提示 **充值/余额不足** 并说明最低线。  
3. **高级会员状态**：使用高级型与大模型 → **按量扣费**；余额跌至 **低于最低线** → **停止** 高阶/按量，普通型仍可用（订阅有效前提下）。

## 6.3 余额提现（概要）

用户申请 → 系统/运营 **核算应扣未扣** → 扣减后 **提出剩余余额** → 更新账本与订单状态（细则见二册与后续 `doc/logic/`）。

## 6.4 后台运营

管理员维护产品与计费配置 → 监控 **水位与应扣未扣** → 处理订阅/提现/风控 → 复盘数据。
